# Domains where WebFetch is known to fail. Format: domain[|guidance].
# Subdomains match. !domain = exception, must come before the block it carves out.
# Full strategy map: fetch-blocked skill (dotagents/skills/fetch-blocked/SKILL.md).
linkedin.com|LinkedIn requires login. Use agent-browser --headed (see LinkedIn section in global instructions).
x.com|Single post: use the anonymous syndication endpoint (invoke the fetch-blocked skill). Search: /x-search. Profiles/threads: agent-browser --headed.
twitter.com|Single post: use the anonymous syndication endpoint (invoke the fetch-blocked skill). Search: /x-search. Profiles/threads: agent-browser --headed.
instagram.com|Instagram requires login. Use agent-browser --headed.
reddit.com|WebFetch cannot reach reddit at all. Use httpie against old.reddit.com: HTML works anonymously, or append .rss for structured Atom (.json is 403). See the fetch-blocked skill.
stackoverflow.com|WebFetch cannot reach Stack Overflow. Plain httpie works (http GET <url>), or use the Stack Exchange API for structured JSON. See the fetch-blocked skill.
nytimes.com|WebFetch cannot reach nytimes.com. Plain httpie works (paywall still applies to full articles).
amazon.com|WebFetch gets a 500 bot block. Use httpie with a browser User-Agent header.
amazon.co.jp|WebFetch gets a 500 bot block. Use httpie with a browser User-Agent header.
imdb.com|WebFetch gets an empty AWS WAF challenge. Star rating: POST to caching.graphql.imdb.com (ratingsSummary.aggregateRating). Name→ID: suggestion endpoint (v3.sg.media-imdb.com/suggestion/x/<query>.json). See the fetch-blocked skill.
naver.com|WebFetch refuses naver client-side. Plain httpie works (http GET <url>) — server-rendered HTML, no browser UA needed.
quora.com|Bot-walled (403 even to httpie). Use agent-browser --headed, only if really needed.
glassdoor.com|Bot-walled (403 even to httpie). Use agent-browser --headed, only if really needed.
